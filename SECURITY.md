# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a vulnerability

Email **security@krishyaid.dev** with:

- A description of the issue
- Steps to reproduce
- Impact assessment if known

We aim to acknowledge reports within 72 hours. Please do not open public GitHub issues for undisclosed security problems.

## Scope

Reports are in scope when they affect Sluice's ability to enforce policy, leak audit data unexpectedly, or bypass carryover memory checks.

Out of scope: issues in upstream MCP servers, client applications, or deployment misconfiguration outside Sluice defaults.
