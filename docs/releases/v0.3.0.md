# Sluice v0.3.0

**The MCP gate that remembers what your agent already saw.**

Sluice sits between AI agents (Cursor, Claude Desktop, Cline) and MCP tool servers. It scans traffic for secrets and PII, and blocks cross-tool leaks when a sensitive value from an earlier tool response shows up in a later tool call.

## Install

```bash
pip install sluice-taint
sluice init
sluice serve
```

PyPI: https://pypi.org/project/sluice-taint/0.3.0/

## Headline feature: session taint

```
read_file  → response contains AKIA...        ✓ allowed (first sighting)
send_email → body contains same AKIA...       ✗ blocked (taint_leak)
```

Run the demo: `bash scripts/demo.sh`

## What's in v0.3.0

### Core (v0.1)
- stdio + HTTP + streamable HTTP transports
- Secret, PII, and tool-poisoning detectors
- Session-scoped carryover memory (`taint_leak`)
- Per-upstream / per-tool YAML policy
- SQLite audit log and `sluice logs` CLI
- `${VAR}` config expansion and legacy config migration

### Credibility (v0.2)
- Policy presets: `filesystem`, `github`, `slack`, `postgres`, `brave-search`
- Read-only HTML dashboard at `/_sluice/`
- Docker image and `docker compose` quickstart
- Integration tests in CI

### Differentiator (v0.3)
- **Taint v2** — JSON-path provenance and propagation graph in audit log
- **Prompt-injection** detector on tool responses
- **OpenTelemetry** exporter (`pip install sluice-taint[otel]`)
- Full streamable HTTP SSE reconnect with `Last-Event-Id`
- Plugin docs and detector catalogue

## Quick start — Cursor / Claude Desktop

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "sluice",
      "args": ["stdio", "--config", "/absolute/path/to/config.yaml"]
    }
  }
}
```

## Docker (HTTP mode)

```bash
docker run --rm -p 4444:4444 \
  -v "$PWD/config.yaml:/etc/sluice/config.yaml:ro" \
  ghcr.io/krishyaid-coder/sluice:0.3.0 serve --config /etc/sluice/config.yaml --host 0.0.0.0
```

## Performance
